## 系统当前可识别的命名规则（以文件名主体为准）

说明：系统会尽量从文件名中识别出「学号」「姓名」「班级」「作业名称」，其中「学号、姓名」是最关键字段；其余字段缺失不影响批改，但会影响导出表的可读性与排序体验。

### 1) 推荐标准格式（最稳定）

```
班级+姓名+学号+作业名称
```

示例：

```
25计算机科学与技术1班+张三三+202502210111+职业规划书.docx
```

### 2) 常见分隔符格式（系统支持）

系统支持将下列分隔符视为等价：`+`、`_`、`-`、`@`、`|`、`｜`、`.`、空格。

示例：

```
25计算机科学与技术1班-张三三-202502210111-专业分析报告.docx
25计算机科学与技术1班_张三三_202502210111_专业分析报告.docx
202502210111 张三三 25计算机科学与技术1班 专业分析报告.docx
```

### 3) 关键字段识别规则（系统策略）

- 学号：会优先识别文件名中「6～20 位连续数字」作为学号（取最长的一段）。
- 姓名：支持中文姓名（2～10 个中文字符）或英文名（字母+连接符）。
- 班级：优先识别包含“班”字或包含 “class” 的片段作为班级。
- 作业名称：优先取文件名尾部、且不属于班级/姓名/学号的片段作为作业名称。

### 4) 键值对格式（系统支持）

示例：

```
班级=25计算机科学与技术1班_姓名=张三三_学号=202502210111_作业=专业分析报告.docx
Class-25计科1_Name-张三三_ID-202502210111_Work-专业分析报告.docx
```

### 5) 无分隔符极端格式（尽力支持）

示例：

```
25计算机科学与技术1班张三三202502210111专业分析报告.docx
```

说明：该类格式依赖“学号数字段”作为锚点进行拆分，若姓名/班级与其他文本混在一起过于复杂，识别可能失败；建议改用第 1) 或 2) 方案。
